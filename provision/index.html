<h1 id="device-provisioning-and-fleet-management-api">Device Provisioning and Fleet Management API</h1> <p>Both APIs use the HTTP POST, GET, PUT and DELETE methods to configure the different facets of the provisioning system. Note that the terms &quot;client&quot; and &quot;device&quot; are interchangeable in the text below. There are various HTTP required headers that are inherent to the protocol that are not repeated in each API procedure description. As an example, all of the POST and PUT requests must also contain a correct <code>Content-Length</code> header specifying the body payload size. Also, all responses having a status of 200 will also contain a <code>Content-Length</code> response header.</p> <p>API Version: <code>1.1</code></p> <p>Host Address: <code>http://m2.exosite.com</code> or <code>https://m2.exosite.com</code></p> <p>If you&#39;re completely new to Exosite&#39;s APIs, you may want to read the <a href="../README.md">API overview</a> first.</p> <h2 id="libraries-and-sample-code">Libraries and Sample Code</h2> <p>Wrapper libraries for this API:</p> <ul> <li>Python: <a href="https://github.com/exosite-labs/pyonep">pyonep</a> includes a module, provision.py, that wraps the provision API.</li> <li>.NET: <a href="https://github.com/exosite-labs/clronep">clronep</a> includes Provision.cs that wraps the provision API.</li> <li>C++: <a href="https://github.com/exosite-labs/cpponep">cpponep</a> includes Provision.cpp that wraps the provision API.</li> <li>Go: <a href="https://github.com/exosite-labs/goonep">goonep</a> includes provision.go that wraps the provision API.</li> </ul> <p>Code samples that use this API:</p> <ul> <li><a href="https://github.com/exosite-garage/provision_examples">Provision Examples</a> - socket level code intended as a reference for implementation in other languages</li> </ul> <h2 id="conventions">Conventions</h2> <p>This document uses a few notational conventions:</p> <ul> <li>A name in angle brackets, e.g. <code>&lt;myvar&gt;</code>, is a placeholder that will be defined elsewhere.</li> <li>Items in curly braces, e.g., <code>{vendor=&lt;id&gt;}</code>, are optional.</li> <li><code>...</code> indicates the previous content is repeated.</li> </ul> <h2 id="http-responses">HTTP Responses</h2> <p>Typical HTTP response codes include:</p> <table> <thead> <tr> <th style="text-align:left">Code</th> <th style="text-align:left">Response</th> <th style="text-align:left">Description</th> </tr> </thead> <tbody> <tr> <td style="text-align:left">200</td> <td style="text-align:left">OK</td> <td style="text-align:left">Successful request, returning requested values</td> </tr> <tr> <td style="text-align:left">204</td> <td style="text-align:left">No Content</td> <td style="text-align:left">Successful request, nothing will be returned</td> </tr> <tr> <td style="text-align:left">4xx</td> <td style="text-align:left">Client Error</td> <td style="text-align:left">There was an error with the request by the client</td> </tr> <tr> <td style="text-align:left">5xx</td> <td style="text-align:left">Server Error</td> <td style="text-align:left">There was an error with the request on the server</td> </tr> </tbody> </table> <h1 id="endpoints">Endpoints</h1> <p>The following procedures allow device OEMs to create custom fleet management applications.
